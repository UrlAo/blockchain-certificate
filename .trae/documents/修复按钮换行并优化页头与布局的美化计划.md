## 问题与原因
- “返回首页”按钮出现换行，主要因为页面内联样式设置了 `input, button { width: 100%; }`，导致页头中的按钮在横向布局中被强制占满整行并换行。
- 页头布局需要更强约束（不换行、右侧状态区不被拉伸），并让按钮在页头中使用自适应宽度。

## 目标
- 消除按钮换行与挤压问题，保证页头在桌面与移动端都美观。
- 优化色彩、间距与徽章呈现，提升整体 user-friendly 体验，但保持零依赖与现有结构。

## 具体改动
### 1) 移除冲突内联样式（issuer/verifier）
- 删除或覆盖页面 `<style>` 中的 `input, button { width: 100%; }` 行，改为：
  - `input { width: 100%; }` 保持表单输入控件宽度
  - 页头按钮使用 `width: auto`（通过全局样式或更具体选择器覆盖）

### 2) 全局样式调整（frontend/styles.css）
- 页头与状态区
  - `.header { display:flex; align-items:center; justify-content:space-between; gap:12px; flex-wrap:nowrap }`
  - `.title { flex:1 }`
  - `.status { display:flex; gap:8px; align-items:center; white-space:nowrap; flex-shrink:0 }`
  - `.badge { max-width:220px; overflow:hidden; text-overflow:ellipsis }`
  - `@media(max-width:480px) { .badge { display:none } }`
- 按钮宽度与上下文
  - `.header .link-btn, .header .btn-primary { width:auto; padding:8px 12px }`
  - `.row button { width:auto }`
  - 表单区仍保持 `.input { width:100% }`
- 微调卡片与间距，统一视觉层次（半径、阴影、间距）

### 3) 页面结构微调
- issuer/verifier 页头保持一致，删除旧的 `walletInfo` 块，仅使用徽章 `badge` 展示状态（地址与 chainId），避免多余换行。
- 验证页将样式引入的 `<link>` 保持在 `<style>` 外，避免解析错误（已修复）。
